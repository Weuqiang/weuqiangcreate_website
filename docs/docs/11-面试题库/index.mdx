## 🎯 使用建议

### 1. 按模块学习
根据面试岗位选择对应模块：
- **前端岗位**：前端面试题 + 算法面试题
- **后端岗位**：后端面试题 + 算法面试题 + 系统设计
- **全栈岗位**：前端 + 后端 + 算法 + 系统设计

### 2. 理解原理
- ❌ 不要死记硬背答案
- ✅ 理解背后的原理
- ✅ 能够举一反三
- ✅ 用自己的话解释

### 3. 动手实践
- 所有代码题都要自己实现一遍
- 在LeetCode上刷题验证
- 总结常见题型和解题模板

### 4. 总结归纳
- 整理自己的答题模板
- 记录易错点和注意事项
- 定期复习巩固

## 💡 答题技巧

### STAR法则
适用于项目经验和行为面试：
- **S**ituation：情境 - 描述背景
- **T**ask：任务 - 说明目标
- **A**ction：行动 - 讲述过程
- **R**esult：结果 - 展示成果

### 技术问题答题框架
1. **先总后分**：先给出核心答案，再展开细节
2. **举例说明**：用实际项目经验支撑答案
3. **对比分析**：对比不同方案的优缺点
4. **双向沟通**：适当提问，展现思考能力

### 算法题解题步骤
1. **理解题意**：确认输入输出，询问边界条件
2. **分析思路**：说出解题思路，不要直接写代码
3. **编写代码**：边写边解释，注意代码规范
4. **测试验证**：用测试用例验证，考虑边界情况
5. **优化分析**：分析时间空间复杂度，讨论优化方案

## 💪 备考建议

1. **提前准备**：至少提前1-2个月开始准备
2. **系统学习**：按模块系统学习，不要东一榔头西一棒
3. **多做题**：算法题至少刷100+道
4. **模拟面试**：找朋友或使用在线平台模拟
5. **总结复盘**：每次面试后总结经验教训
6. **保持自信**：相信自己，展现最好的状态
