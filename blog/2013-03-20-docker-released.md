Docker 正式发布，提出了容器化的概念。这会改变软件部署的方式。

<!-- truncate -->

## 什么是 Docker

Docker 是个容器化平台：

- 把应用和依赖打包在一起
- 在任何环境都能运行
- 比虚拟机轻量

## 解决了什么问题

**"在我机器上能跑"**

以前经常遇到：

- 开发环境能跑，生产环境不行
- 依赖版本不一致
- 环境配置复杂

Docker 解决了这些问题。

## 核心概念

**镜像（Image）**

应用的模板，包含代码和依赖。

**容器（Container）**

镜像的运行实例。

**仓库（Registry）**

存储镜像的地方，类似 GitHub。

## 对比虚拟机

| 特性 | Docker | 虚拟机 |
|------|--------|--------|
| 启动速度 | 秒级 | 分钟级 |
| 资源占用 | 低 | 高 |
| 隔离性 | 进程级 | 系统级 |

Docker 更轻量，但隔离性稍弱。

## 对行业的影响

Docker 的出现：

- 推动了微服务架构
- 改变了部署方式
- 催生了 Kubernetes

现在，容器化已经是标配。

## 争议

**1. 安全性**

容器共享内核，隔离性不如虚拟机。

**2. 学习曲线**

概念多，新手不好上手。

## 总结

Docker 是个革命性的工具：

- 解决了环境一致性问题
- 推动了 DevOps 发展
- 改变了软件部署方式

2013 年，容器化时代开始了。
